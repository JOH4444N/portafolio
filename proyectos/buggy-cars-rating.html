Descripción del proyecto

Automatización de pruebas funcionales sobre Buggy Cars Rating con Cypress 15 — 8 historias de usuario, más de 50 casos de prueba y pipeline CI/CD completo sobre Docker + Jenkins.

Page Object Model & Fixtures

Se aplicó el patrón Page Object Model para separar la lógica de interacción de los casos de prueba. Cada pantalla tiene su propia clase que encapsula selectores y métodos, haciendo la suite más mantenible y escalable. Los fixtures centralizan todos los datos de prueba.

Page Objects Fixtures Separación de responsabilidades Mantenibilidad
Custom Commands

Se crearon custom commands (cy.login(), cy.logout(), cy.navegarA()) para encapsular acciones reutilizables a lo largo de toda la suite. Esto reduce duplicación de código y hace los tests más legibles, describiendo flujos complejos en pocas líneas.

cy.login() cy.logout() cy.navegarA() DRY principle
Pruebas de integración con cy.intercept()

En los flujos que involucran comunicación con el backend se usó cy.intercept() para validar método HTTP, cuerpo del request y código de respuesta. Esto garantiza que los contratos entre frontend y API se cumplan correctamente en cada ejecución.

cy.intercept() HTTP validation Request body Status codes
CI/CD — Jenkins + Docker

Infraestructura sobre Ubuntu 24.04 con Docker Engine. Jenkins corre en un contenedor con socket binding al host, permitiendo levantar contenedores hermanos desde el pipeline. Las pruebas se ejecutan en un contenedor efímero basado en cypress/included:15.9.0, montando el workspace como volumen compartido.

Ubuntu 24.04 Docker Engine Jenkins container Socket binding cypress/included:15.9.0
Reportes Mochawesome & GitHub Pages

El pipeline declarativo automatiza el ciclo completo: instalación, ejecución, generación y publicación del reporte. Los resultados se procesan con Mochawesome y el HTML se despliega en GitHub Pages desde una rama dedicada tras cada ejecución. Un bloque try/catch garantiza que el reporte se publique aunque fallen los tests, marcando el build como FAILURE.

BUG-001 Falla en cierre de sesión desde módulo Overall Rating Autenticación
Resultado esperado

El sistema debe cerrar la sesión correctamente desde cualquier módulo. La barra de navegación debe mostrar la opción "Login" y no debe visualizarse el mensaje "Hi, username".

Resultado actual

El sistema presenta un comportamiento inconsistente. Aunque en otros módulos el logout funciona correctamente, en el módulo "Overall Rating" el usuario permanece autenticado después de hacer clic en "Logout".

BUG-002 Falta de redirección después de Logout desde diferentes módulos Autenticación
Resultado esperado

El sistema debe redirigir automáticamente al usuario a la página principal después de cerrar sesión desde cualquier módulo.

Resultado actual

El sistema no redirige al usuario después de hacer logout, permaneciendo en el módulo actual independientemente desde dónde se ejecute la acción.

BUG-003 Campo Age permite caracteres no numéricos Perfil
Resultado esperado

El sistema debe permitir únicamente valores numéricos enteros en el campo Age y rechazar cualquier otro tipo de carácter.

Resultado actual

El sistema permite ingresar caracteres alfabéticos en el campo Age, evidenciando una falta de validación del tipo de dato.

BUG-004 Campo Gender no es lista desplegable Perfil
Resultado esperado

El campo Gender debe mostrarse como una lista desplegable (dropdown) con opciones predefinidas para seleccionar.

Resultado actual

El campo Gender se muestra como un input de texto, permitiendo al usuario ingresar valores manualmente en lugar de seleccionar de una lista.

BUG-005 Reutilización de contraseña actual permitida Password
Resultado esperado

El sistema debe validar que la nueva contraseña sea diferente a la actual, mostrando un mensaje de error y evitando la actualización.

Resultado actual

BUG-006 Validación incorrecta de longitud mínima en el campo New Password Password

Resultado esperado

El sistema no debe permitir el cambio. El botón "Save" debe permanecer deshabilitado y mostrar un mensaje claro indicando los campos obligatorios faltantes.

Resultado esperado
