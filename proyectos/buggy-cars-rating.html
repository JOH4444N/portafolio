Descripción del proyecto

Proyecto de automatización de pruebas funcionales sobre la plataforma Buggy Cars Rating, desarrollado con Cypress 15. Se implementaron 8 Historias de Usuario con más de 50 casos de prueba cubriendo los flujos de registro, autenticación, gestión de perfil, navegación, votación y filtrado de automóviles.

Se aplicó el patrón Page Object Model para separar la lógica de interacción de los casos de prueba, fixtures para centralizar los datos de prueba, y custom commands (cy.login(), cy.logout(), cy.navegarA()) para encapsular acciones reutilizables a lo largo de la suite. En los flujos que involucran comunicación con el backend se utilizaron pruebas de integración mediante cy.intercept(), validando método HTTP, cuerpo del request y código de respuesta.

La infraestructura de ejecución está montada sobre una máquina virtual Linux (Ubuntu 24.04) con Docker Engine instalado. Jenkins corre dentro de un contenedor Docker con acceso al socket del host mediante socket binding, permitiendo levantar contenedores hermanos desde el pipeline. Las pruebas se ejecutan en un contenedor efímero basado en la imagen oficial cypress/included:15.9.0, montando el workspace como volumen compartido.

El pipeline declarativo en Jenkins automatiza el ciclo completo: instalación de dependencias, ejecución de pruebas, generación de reporte y publicación. Los resultados se procesan con Mochawesome y el reporte HTML se despliega automáticamente en GitHub Pages desde una rama dedicada, quedando accesible públicamente tras cada ejecución.

El pipeline implementa manejo de errores con try/catch garantizando que el reporte se publique independientemente del resultado de los tests, marcando el build como FAILURE cuando alguna prueba falla.
