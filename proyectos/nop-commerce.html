nopCommerce – Manual QA Testing

Proyecto de pruebas manuales sobre una plataforma e-commerce basado en criterios de aceptación.

Volver al portafolio

Descripción del proyecto

Proyecto de aseguramiento de la calidad sobre la plataforma nopCommerce, enfocado exclusivamente en pruebas manuales. Se diseñaron y ejecutaron 160 casos de prueba distribuidos en 5 épicas y 11 historias de usuario, validando los principales procesos funcionales mediante pruebas funcionales, de regresión y exploratorias.

Estrategia de pruebas

La estrategia de pruebas se definió tomando como base los criterios de aceptación de cada historia de usuario. Se ejecutaron pruebas funcionales, pruebas de regresión y sesiones de pruebas exploratorias para validar el correcto funcionamiento de los procesos críticos del sistema.

Diseño de Casos de Prueba

Los casos de prueba fueron elaborados a partir de las historias de usuario y sus criterios de aceptación, cubriendo escenarios positivos, negativos, validaciones de reglas de negocio y manejo de errores para garantizar una adecuada cobertura funcional.

Gestión de Defectos

Durante la ejecución de las pruebas se registraron los defectos encontrados, documentando pasos para reproducir, resultado esperado, resultado obtenido, severidad, prioridad y evidencia correspondiente para facilitar su análisis y posterior corrección por parte del equipo de desarrollo.

Cobertura Funcional

El proyecto contempló la validación funcional de los módulos más importantes de la plataforma nopCommerce, incluyendo el registro de usuarios, autenticación, catálogo de productos, carrito de compras y proceso de checkout, garantizando el cumplimiento de los criterios de aceptación.

Historias de Usuario

El documento contiene las 11 historias de usuario organizadas en 5 épicas, junto con sus respectivos criterios de aceptación, los cuales sirvieron como base para el diseño de los casos de prueba.

Plan de Pruebas

El plan de pruebas describe el alcance del proyecto, objetivos, estrategia de pruebas, tipos de prueba ejecutados, criterios de entrada y salida, riesgos identificados, entre otros.

BUG-001 No existe acceso visible al módulo de comparación de productos Navegación
Resultado esperado

El sistema debe permitir acceder correctamente al módulo de comparación de productos mediante una opción visible y accesible desde la interfaz principal.

Resultado actual

No se encontró una opción visible para acceder al módulo de comparación de productos desde la interfaz del sistema.

BUG-002 Presencia de enlace de noticias en la sección de redes sociales del footer Footer
Resultado esperado

El sistema debe mostrar únicamente enlaces correspondientes a redes sociales oficiales dentro de la sección de redes sociales del footer, redirigiendo correctamente a sus respectivas páginas.

Resultado actual

Los enlaces de redes sociales funcionan correctamente; sin embargo, se identificó un enlace de noticias dentro de la sección, el cual no corresponde a una red social.

BUG-003 Registro exitoso con contraseña sin símbolos especiales Registro
Resultado esperado

El sistema no debe permitir el registro si la contraseña no cumple las reglas de complejidad definidas, incluyendo el uso de símbolos especiales.

Resultado actual

El sistema permitió realizar el registro exitosamente con una contraseña que no contiene símbolos especiales.

Resultado esperado
