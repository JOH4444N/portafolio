Plan de Pruebas

Objetivos

  • Validar funcionalidades críticas del e-commerce
  • Identificar defectos en flujos principales
  • Verificar usabilidad y experiencia de usuario
  • Asegurar compatibilidad cross-browser

Alcance

Las pruebas cubrieron los módulos principales: autenticación, catálogo de productos, carrito de compras, proceso de pago, gestión de pedidos y panel de usuario.

Casos de Prueba Ejecutados

TC001 - Registro de Usuario

Objetivo: Verificar el registro exitoso de nuevos usuarios

  1. Navegar a la página de registro
  2. Completar formulario con datos válidos
  3. Hacer clic en "Registrarse"
  4. Verificar email de confirmación

Resultado esperado: Usuario registrado exitosamente

PASÓ

TC015 - Agregar Producto al Carrito

Objetivo: Validar la funcionalidad de agregar productos

  1. Seleccionar un producto del catálogo
  2. Elegir cantidad y variantes
  3. Hacer clic en "Agregar al carrito"
  4. Verificar actualización del contador

Resultado esperado: Producto agregado correctamente

FALLÓ

Bug Reportado: BUG-001

Descripción: El contador del carrito no se actualiza inmediatamente al agregar productos

Severidad: Media

Pasos para reproducir:

  1. Agregar cualquier producto al carrito
  2. Observar el contador en la esquina superior
  3. El contador permanece en 0 hasta refrescar la página

TC032 - Proceso de Checkout

Objetivo: Verificar el flujo completo de compra

  1. Acceder al carrito con productos
  2. Proceder al checkout
  3. Completar información de envío
  4. Seleccionar método de pago
  5. Confirmar pedido

Resultado esperado: Pedido procesado exitosamente

Principales Hallazgos

  • Funcionalidad crítica: El proceso de pago funciona correctamente
  • Problema de UX: Varios elementos de interfaz no son intuitivos
  • Performance: Tiempos de carga lentos en páginas de producto
  • Compatibilidad: Issues menores en navegadores Safari

Recomendaciones

  1. Prioridad Alta: Corregir bugs críticos en el carrito de compras
  2. Prioridad Media: Mejorar feedback visual en formularios
  3. Prioridad Baja: Optimizar tiempos de carga de imágenes
  4. Futuro: Implementar pruebas automatizadas para regresión
